设计模式（一）：从原型到继承

发表于 2019-03-20 | 更新于 2019-03-21 | 分类于 javascript

面向对象的设计就是将需求抽象成一个对象，然后针对把共享相同结构和行为的对象的集合称为”类”。小红书里对创建对象提出了几种常见的设计模式，这些模式都围绕了”构造”和”原型”两个重要的概念来给对象添加方法。

今天，就先从原型和继承看一下吧。首先，什么是设计模式呢？

设计模式

设计模式是什么

设计模式是软件开发中针对常出现的，共性的问题提出的可复用的方案。

Design patterns are reusable solutions to commonly occurring problems in software design.[^1]

设计模式需要解决一类问题，是一种对关系的描述：

  1. a context
  2. a system of forces in that context
  3. a configuration that allows these forces to resolve themselves in context

设计模式分类

GoF一共提出了23种oop的设计模式，并这些模式分为三类。ES5对js引入了类的原生支持，但都不过是基于原型继承而模拟的。

Creational创造型

核心在于把控（control）创建对象的过程，从而简化项目复杂度

Based on the concept of creating an object.

  • Factory 工厂：创造一个有多个派生类的实例
  • Abstract Factory 抽象工厂：创造一个拥有很多父类的实例
  • Prototype 原型：一个可被复制的已经的初始化实例
  • Singleton 单例：该类只允许存在一个实例，该实例可被全局访问
  • Builder 建造：只创建同一类型的对象，该对象的构造与表达分离

Structural结构型

重点是拆分不同对象的关系，从而当整个系统中某一个部分改变时，系统的结构不需要改变

Based on the idea of building blocks of objects.

  • Adapter 适配器：适配不同类的界面，从而可以兼容地一起工作
  • Bridge 桥：分离对象的界面和实现，这是桥断模式吧
  • Decorator 装饰器：动态给对象添加处理方法
  • Facade 外观 哈哈立面模式建筑🐶小声说 ：一个把整个子系统隐藏起来的单类
  • Flyweight 轻量：一个非常精细的实例，可以高效地共享其它地方的信息
  • Composite 组合：一个结构，实现1＋1>2
  • Proxy 代理：真正对象的一个代理对象

Behavioral行为型

目的在于优化系统内部各个独立对象的交流或者流线化这个交流过程

Based on the way objects play and work together.

  • Chain of Responsibility 责任链：把一个请求传递在这个对象的链上直到找到能肩负起这个请求的那个对象
  • Command 命令：把一个请求或者操作封装到一个对象中
  • Memento 备忘：记录对象的内部状态以备之后还原
  • State 状态：对象状态改变时让它行为也改变
  • Strategy 策略：将一个算法封装在类里面，把调用和实现分离
  • Iterator 迭代器：按顺序访问集合collection里的各元素
  • Mediator 调解者：阻止类之间的显式引用
  • Observer 观察者：如果观察到了改变就去通知这些类们
  • Visitor 访问者：给这个类悄摸摸地添加一个新操作

没有写过项目还是比较抽象的。其实很多概念都理解的不到位，所以还是从最最简单的地方重新开始吧。只做概念理解，不展开讲基础。

复习下基础知识吧

对象

对象这个翻译本身就比较抽象，相比之下，object就好理解多了，就是一个独立的个体罢了。这个个体拥有一些属性和自己的类型，也就是它能干什么，它是个什么类型。

…an object is a standalone entity, with properties and type.[^2]

object是ES6中有7个基本类型之一。

“类”

在其它语言中，class是一个集合，该集合里是拥有一些共性的行为的object。ES5中为js引入class关键字，但其实是基于原型继承的一个特殊函数。

函数

在js中，函数是first class object，它们和其它object一样有自己的属性properties和方法methods。其实就是变量和函数，当它们属于一个对象时叫属性和方法而已。

区别在于函数可以被called，也就是调用。可以在函数上使用new关键词创建一个新的object。

在之前的文中，有讲到new的作用机制：

  1. 创建一个新对象，该对象继承自Foo.prototype
  2. 使用指定的参数，调用构造函数 Foo ，并将 this 绑定到新创建的对象。
  3. 由构造函数返回的对象就是 new 表达式的结果。

当时重点在于讲this的作用域所以没有展开讲构造函数。现在再来看一下构造函数是什么。

构造函数

ES6之前

在ES6之前js中的constructor实际上是一个属性：
所有object(除了Object.create(null)创建的)都从它的原型上继承一个 constructor 属性。

而一般所说的构造函数是指所有带new的函数调用[^3]。即，对一个普通函数使用new关键字时，该函数则变成了一个“构造函数”。

ES6中

在ES2015(ES6)中，在class中定义了一个constructor函数如下

The constructor method is a special method for creating and initializing an object created within a class

  1. constructor是object中的一个特殊函数
  2. constructor的作用是创建＋初始化一个新object
  3. 这个新object位于class内部

所以当在ES6中对class使用new关键字创建实例时，会调用class中的constructor函数

更多关于ES6中的class：ECMAScript 6 入门，阮一峰
以下主要针对ES5及之前。

一些小细节

  1. js中有7个基本类型，有9个常用的内置对象(也就是构造函数)：Object(), Array(), String(), Number(), Boolean(), Date(), Function(), Error() ,RegExp()
  2. 创建基本类型时一般使用字面量(js会在后台相应地创建一个临时对象)
  3. 构造函数按惯例首字母大写，但实际上对js引擎而言也是个普通函数

再重新看一下new

让我们再看一下new一个普通函数的时候，到底发生了什么[^4]：

  1. 创建了一个新对象，这个对象的类型 (最开始讲object的时候说过的，每个object都有属性和类型)指向object
  2. 将新对象内部的[[prototype]]属性设置成Foo.prototype (即构造函数的原型属性)
  3. 将this绑定到新对象上
  4. 调用Foo(name)，如果遇到this就使用绑定的新对象
  5. 返回新对象并赋给声明的变量名a；如果Foo函数返回的是非null的object reference，则返回这个引用。

注意这里并没有出现constructor属性，实际上a.constructor === Foo中，.constructor的引用被委托给了Foo.prototype，而Foo.prototype.constructor默认指向Foo。我们最开始就说了：新创建的object会从它的原型上继承一个 constructor 属性，所以这个.constructor实际上是继承来的。


注意：如果我们让Foo.prototype指向新的原型对象，那么这个.constructor在a和Foo中都无法找到.constructor属性，会顺着原型链找到Object.prototype，而指向Object(...)。

插入一个小小小的小结

.constructor并不是一个不可变属性。它是不可枚举的，但是它的值可以被修改的。此外，可以给任意[[prototype]]链中的任意对象添加一个名为constructor的属性或者对其进行修改，可以任意对其赋值。[^3]

所以调用new时最关键的就是第2步：其中内部[[prototype]]是不可见的，而prototype属性是可见可读的。让我们终于来到了今天的重点：原型。

终于来到了原型

先看看prototype

当创建一个函数时，它的prototype属性被按某个规则创建。这个属性包含一个对象，所有实例对象需要共享的属性和方法，都可以放在这里。默认情况下，所有的prototype都会有一个construction属性，这个属性指回这个函数。也就是前文的Foo.prototype.constructor默认指向Foo。之后，其它的属性和方法可以加到prototype上。

那[[prototype]]呢

当构造函数被调用创建新实例时 (也就是第2步)，这个实例会有一个内部指针指向该构造函数的property属性，这个指针被叫做[[prototype]]。在一些浏览器中，可以通过__proto__访问[[prototype]]。通常情况下，这个属性是完全隐藏的。

所以在上面的🌰中，实例a是直接连接到Foo.prototype，而不是Foo本身。实例对象创建时，除了本身自己的属性和方法，也会引用[[prototype]]指向的属性。

[[prototype]]更多地是指这种指向关系，虽然不可访问，但可以通过isPrototypeOf()来判断是否存在。ES5中添加了Object.getPrototypeOf()可以返回[[prototype]]的值。

在一些书里，原型链也叫做[[prototype]]链。所有普通的[[prototype]]链最后都会指向内置的Object.prototype。是不是和前面替换Foo.prototype之后说的一样呢。

让我们跟着课本最后看一遍构造函数，原型和实例的关系：

Each constructor has a prototype object that points back to the constructor, and instances have an internal pointer to the prototype.

今天的重头戏：继承

课本里提供了6种继承方式，再次重申js里的继承主要是依赖原型链来实现的。

原型链继承

重点：将子类的原型改写到父类的实例上

机制(其它继承类型将不再赘述)：

  1. 首先，如果我们调用Super.prototype，它指向一个对象{constructor: f}，该对象拥有一个constructor属性(指向Super())和__proto__属性。
  2. 然后我们添加了新的方法getSuper()
  3. 之后声明了新的函数Sub
  4. 然后我们重写了Sub.prototype 。这里用new关键字创造了一个Super的实例对象，此时，Sub.prototype便拥有了Sub的属性和方法。并且它的__proto__ 将指向Super.prototype。⚠️：getSuper()在Super.prototype中，而prop作为一个实例属性，在Sub.prototype中。
  5. 最后我们创建了一个sub的实例instance。此时，instance.constructor指向的是Sub.prototype.constructor，而Sub.prototype被重写为Super，Super.constructor是自己。如果需要修复则加上Sub.prototype.constructor = Sub

特点：

  1. 子类的原型继承父类的属性，并且可以引用父类原型中的方法；所以父类新增的都可以访问到
    －－－
  2. 子类的实例共享子类原型上的父类属性，即prop
  3. 创建子类实例时，不能在不影响其它实例的情况下向父类传参数

构造继承

重点：使用call，apply把this显式绑定在Sub()实例上

特点：

  1. 只在子类上绑定了父类的属性
  2. 每个子类拥有自己的属性副本，彼此不影响
  3. 可以向父类传参数
  4. 可以继承多个父类
    －－－
  5. 方法都要在构造函数中定义无法复用
  6. 子类不能从原型继承方法属性

组合继承：原型链＋构造

重点：需要继承父类原型的通过原型链，需要拥有本地的父类属性副本的用构造

特点：

  1. 可以从父类原型继承方法属性，可以传参数，可以复用
  2. 每个新实例分别拥有自己从父类继承的属性
    －－－
  3. 调用了两次父类构造函数：line18, line12

原型式继承: object()

“非构造函数”的继承，即让一个普通对象关联另一个普通对象。
重点：把子对象的prototype属性指向父对象，其实是对父对象进行了一次浅复制

特点：

  1. 不需要构造函数，通过对象继承另一个对象
    －－－
  2. 引用类型值属性会被子对象共享，即相当于N个父对象的副本

寄生式继承

重点：给新对象再封装一次增强对象后再返还

特点：

  1. 可以增强对象，添加新方法
    －－－
  2. 使用寄生式继承给对象添加函数，不能做到函数复用而降低效率

寄生组合式继承(最理想的继承范式)

上面提到组合继承最大的问题在于每次都要调用两次父类的构造函数。
重点：构造函数来继承属性，原型链的混成形式来继承方法。在Sub.prototype = new Super()这一步时，其实只是希望有一个父类原型的副本，可以使用寄生式继承来实现

特点：

  1. 高效率，只调用一次sup的构造函数
  2. 原型链保持不变，可以正常使用instanceof, isPrototypeOf()
