其实算不上阶段性的总结和计划

发表于 2019-04-25 | 更新于 2019-04-25 | 分类于 plan

面试完回来收获还是很大的，对目前的发展的小方向有了一些想法，想重新梳理一下思路记在这里。

计算机基础

首先是计算机基础方面，这个书单可以开的没完没了了，但是主要是操作系统，CSAPP，计算机网络，算法和数据结构。

前端相关

css

面试的时候发现css比较一塌糊涂了。一方面其实没有看过css的书只是碎片式的知识点，另一方面自己写的还是太少了。不管是代码组织，具体的写法，还是css的属性值都掌握的不是很好。
看到推荐比较多的是《CSS Mastery》，《CSS 揭秘》，《css权威指南第三版》。

重点不是扣细节和背知识点，而是需要用的时候知道用什么，怎么用，去哪里查。而对于基础的地方要明白为什么这么写。

html

最简单的标签，inline，block这些还需要认真看过规范才行

js

js部分其实没什么好说的，确实需要多看多写。书的话看过的书以前可能理解的也就三四成都不错了。我感觉有些地方理解不到位是计算机基础的锅。所以就简单地放个书单吧，不求多，这几本能看个七七八八就好了：
《JavaScript高级程序设计（第3版）》
《JavaScript语言精粹》
《你不知道的JS》
《ES6 标准入门》: promise, generator, proxy和之后的await

比较值得一提的是设计模式。想把设计模式结合《head first》和《JavaScript设计模式与开发实践》认认真真写一遍。
另外就是《高性能JavaScript》关于优化和性能这块也需要开始认真学了。

其实还有好多好多

node.js
webpack
测试和上线
浏览器优化（问我这个真的超纲了喵喵喵

一点规划

组件

主要是组件化相关的：配色算法，sketch插件和ui组件库。

  • 组件化的代码组织
  • 接口怎么写，如何调用
  • dom结构，事件绑定
    目前的打算是学习bootstrap的源码，如果有其它更合适的再来补充

本来这里有其它想法的但是还是先一步一步来吧，以及想把手里的三个很简单的小项目继续完善下去。从写功能需求和设计开始，毕竟还蛮有趣的。
